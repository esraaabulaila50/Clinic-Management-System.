1.مقدمة عن المشروع
يعتبر نظام إدارة العيادة الطبية تطبيقاً ويب ديناميكياً متكاملاً، صُمم ليعالج مشكلة إدارة السجلات الطبية والمواعيد يدوياً. يوفر النظام بيئة رقمية آمنة ومنظمة تتيح للمسؤولين متابعة المرضى والأطباء وحجز المواعيد بكفاءة عالية، مما يقلل من الأخطاء البشرية ويوفر الوقت.

2. الهيكل التقني والبرمجي (Technical Stack)
تم بناء المشروع باستخدام تقنيات الويب الأساسية لضمان التوافق والأداء العالي:
PHP (Procedural Style): تم استخدام الأسلوب الإجرائي في البرمجة لضمان سهولة قراءة الكود وتتبع المنطق البرمجي (Logic) من قبل المطورين الآخرين.
MySQL: تم تصميم قاعدة بيانات علائقية (Relational Database) لضمان تكامل البيانات (Data Integrity).
Bootstrap 5: تم الاعتماد على إطار العمل Bootstrap لتصميم واجهات مستخدم متجاوبة (Responsive UI) تعمل بكفاءة على الحواسيب والهواتف الذكية.
Sessions & Security: تم استخدام الجلسات (Sessions) لتأمين الصفحات الداخلية ومنع الوصول غير المصرح به.

3. تفاصيل قاعدة البيانات (Database Architecture)
تتكون قاعدة البيانات من 5 جداول مترابطة بعلاقات منطقية:
users: لإدارة حسابات الدخول وتحديد أدوار المستخدمين (Admin/Staff).
departments: لتعريف الأقسام الطبية المتاحة في العيادة.
doctors: لتخزين بيانات الأطباء وربطهم بأقسامهم عبر مفاتيح خارجية (Foreign Keys).
patients: سجل شامل لبيانات المرضى الشخصية.
appointments: الجدول المحوري الذي يربط الطبيب بالمريض ويسجل تاريخ وحالة الموعد.

4. وظائف النظام (System Functionalities)
يغطي النظام كافة متطلبات الـ CRUD الأساسية والمتقدمة:
إدارة المرضى: إضافة مرضى جدد، عرض القائمة، تعديل البيانات، وحذف السجلات.
إدارة الأطباء: ربط الأطباء بالأقسام الطبية المختصة وعرض تخصصاتهم.
نظام المواعيد: حجز موعد جديد من خلال قوائم منسدلة ديناميكية تجلب البيانات من جداول المرضى والأطباء.
لوحة التحكم: واجهة رئيسية تعرض إحصائيات سريعة وتوفر روابط وصول سريعة لكافة أقسام النظام.

5. ملاحظات فنية هامة (Technical Observations)
تم مراعاة المعايير التالية أثناء البرمجة لضمان جودة المشروع:
فصل الاتصال: تم عزل كود الاتصال بقاعدة البيانات في ملف مستقل (db.php) لتسهيل عملية التعديل والصيانة وتطبيق مبدأ (Don't Repeat Yourself).
أمان المدخلات: تم استخدام دوال التحقق من المدخلات لمنع الهجمات البرمجية الشائعة وضمان صحة البيانات قبل تخزينها.
تنسيق الرسائل: تم استخدام مكونات Bootstrap (Alerts & Badges) لتظهر رسائل النظام (مثل رسائل الخطأ أو نجاح الإضافة) بشكل احترافي وواضح للمستخدم.
تكامل العلاقات: تم استخدام استعلامات SQL المتقدمة (Inner Join) لعرض البيانات المترابطة بشكل نصي مفهوم بدلاً من عرض أرقام المعرفات (IDs).

6. تعليمات التشغيل
تأكد من تشغيل سيرفر Apache و MySQL من لوحة تحكم XAMPP.
قم بإنشاء قاعدة بيانات باسم clinic_db واستورد ملف clinic_db.sql.
انسخ مجلد المشروع إلى htdocs.
افتح المتصفح على الرابط: localhost/clinic_project/login.php
